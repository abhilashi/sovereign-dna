## 2024-05-24 - Interactive Component Accessibility (Card)
**Learning:** When using generic container elements (like `div` or Framer Motion's `motion.div`) as interactive, clickable components (like Cards), relying solely on `onClick` and `cursor-pointer` ignores keyboard users and screen readers. Extending `HTMLMotionProps` or similar generic React props interfaces can lead to build errors due to type conflicts (e.g. `onDrag`).
**Action:** When creating custom interactive containers like `<Card onClick={...}>`, explicitly add `role="button"`, `tabIndex={0}`, an `onKeyDown` handler that triggers on "Enter" or " " (Space), and use Tailwind's `focus-visible` utilities (e.g., `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent`) to provide a clear focus indicator without breaking pointer interactions. Avoid blanket extending React component interfaces to prevent TypeScript errors.
